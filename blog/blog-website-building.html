Building My Personal Website with GitHub Pages and Cloudflare

Published on: October 16, 2024

Creating my personal website has been an exciting journey—one that taught me valuable lessons in web hosting, DNS management, and cloud services. From setting up the site on GitHub Pages to optimizing it with Cloudflare, the process was a great mix of creativity and technical problem-solving. In this post, I’ll share how I set up my site, overcame challenges, and optimized it for speed and security.

Why I Chose GitHub Pages and Cloudflare

I wanted to create a personal portfolio that showcased my research in bioinformatics and my projects. GitHub Pages was a natural choice because it's free, integrates seamlessly with version control, and is incredibly easy to deploy. On top of that, I decided to use Cloudflare to improve performance and security. Cloudflare’s CDN (Content Delivery Network) and SSL services are ideal for making sure my site is both fast and secure across the globe.

Step 1: Setting Up GitHub Pages

Setting up the website on GitHub was straightforward. Here's a quick overview of how I did it:

  • Create a GitHub Repository: I set up a new repository and named it after my domain: vaibhavkulkarni.bio.
  • Push the Website Code: After coding the basic HTML, CSS, and JavaScript, I pushed everything to the main branch.
  • Enable GitHub Pages: Under the repository settings, I enabled GitHub Pages by selecting the branch I wanted to serve as the source. GitHub Pages took care of the rest and deployed my site at https://vaibhavkulkarni.github.io.

Step 2: Integrating a Custom Domain

The next step was to link my custom domain, vaibhavkulkarni.bio, to the GitHub Pages site. This was an important part of my project, as having a personal domain adds a layer of professionalism and branding to the website.

To do this, I:

  • Added a CNAME file in the root directory of my repository that contained vaibhavkulkarni.bio.
  • Updated the DNS records on my domain registrar’s control panel, setting the CNAME to point to vaibhavkulkarni.github.io.

After a few minutes, my site was live on the custom domain, but I noticed something was missing—HTTPS.

Step 3: Optimizing with Cloudflare

This is where Cloudflare comes in. I wanted to secure my site with SSL encryption and boost its performance using Cloudflare’s CDN. Here's how I did it:

  • Add Domain to Cloudflare: I signed up for a Cloudflare account and added vaibhavkulkarni.bio to their platform. Cloudflare provided me with two nameservers that I updated in my domain registrar’s dashboard.
  • Enable SSL: I went to Cloudflare’s SSL/TLS settings and selected the Full option, which ensures end-to-end encryption between the user and GitHub Pages. Cloudflare automatically issued an SSL certificate for my domain.
  • Performance Enhancements: I enabled Cloudflare’s CDN, which caches content across their global network of servers. This dramatically improved my site’s load time, especially for visitors from regions far from my hosting server.

Step 4: Overcoming Challenges

While the process was generally smooth, I did encounter a few hiccups:

  • SSL Propagation Delay: After setting up Cloudflare’s SSL, it took a few hours for the HTTPS certificate to propagate fully. During that time, accessing the site over HTTPS led to an error. I had to be patient and monitor the progress until the SSL was fully deployed.
  • DNS Cache Issues: After updating the DNS settings, I noticed that my site wasn't resolving correctly on some devices. Clearing the local DNS cache solved this problem, but I had to keep that in mind for future updates.

Lessons Learned

This experience was not just about setting up a personal website; it was also an opportunity to learn more about web technologies, particularly hosting, DNS management, and web security.

  • Always Enable SSL: In today’s web environment, security is critical. Having HTTPS not only protects your visitors but also improves your site’s ranking on search engines.
  • Use a CDN for Speed: Cloudflare’s CDN made a huge difference in the performance of my site. It ensures faster load times for visitors from different parts of the world, without me needing to configure complex server settings.
  • Monitor DNS Changes: DNS propagation can take time, and issues like SSL errors or DNS cache mismatches can be frustrating. Understanding how these systems work made the troubleshooting process much easier.

Conclusion

Setting up my personal website was both rewarding and educational. GitHub Pages made it easy to deploy and maintain the site, while Cloudflare enhanced its security and performance. Going forward, I plan to continually update the site with my latest projects and blog posts, while keeping it fast and secure.

If you're looking to build your own website, I highly recommend exploring GitHub Pages and Cloudflare. The combination of free hosting and enterprise-level performance tools is hard to beat.
